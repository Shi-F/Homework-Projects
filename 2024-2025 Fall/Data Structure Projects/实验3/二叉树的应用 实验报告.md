## 设计目标

在C++环境下，利用二叉树与Huffman树的相关操作，实现对任意文件的压缩处理，压缩之后的文件后缀名为huff，同时可以对所有后缀名为huff的压缩文件进行解压缩。

以1个字节为单位进行Huffman编码，要求可以完成压缩和解压的步骤，并且解压出来的文件没有任何损失。

## 算法设计

压缩文件时执行的操作： 

1. 读入文件，统计每种字节出现的次数（256种） 
2. 根据字节出现的次数创建Huffman树
3. 由Huffman树得到各字节的Huffman编码
4. 将压缩前文件类型、各字节出现的次数信息写入.huff文件
5. 读入文件，按字节将对应的编码写入.huff文件，缺位用0补齐

解压缩文件时执行的操作：

1. 读入.huff文件，根据文件开头的信息获得Huffman树 
2. 根据Huffman编码，将对应的字节写入文件

.huff文件结构为：

```c++
char FileType[FILE_TYPE_LENGTH] //源文件类型（,xxx） 
int FileSize //源文件大小（用于解压时确定是否解码结束） 
int count[256] //各字节按顺序的出现次数 
//Huffman码比特流 
```

基于贪心策略构建Huffman树的算法思想概括如下：

1. 给定n个权值作为n个叶子节点，构成初始的森林F={T1, T2, ..., Tn}，其中每棵二叉树Ti仅有一个权值为Wi的根节点。
2. 在森林F中选取两棵根节点权值最小的树作为左右子树，构造一棵新的二叉树，并置新二叉树根节点的权值为其左右子树上根节点的权值之和。
3. 从森林F中删除这两棵被选中的树，同时将新构造的二叉树加入到森林F中。
4. 重复步骤2和步骤3，直到森林F中只含一棵二叉树为止，即得到Huffman树。

### 数据结构

采用结构体数组FileByteInfo[256]存储各字节（无符号字符）的信息，下标即表示无符号字符的值，结构体定义为：

```c++
typedef struct Info{
	int count;	//无符号字符出现的次数
	char *HCode;	//字符对应的Huffman编码 
}Info;	//文件内无符号字符的信息
```

Huffman编码过程借助Huffman树实现，Huffman树采用顺序存储，其结点结构定义为：

```c++
typedef struct HTNode{
	unsigned char data;
	int weight;		//权重
	int lchild,rchild;
	int parent;
}HTNode,*HuffmanTree;
```

## 调试分析

测试数据Input.txt采用样例文件：2_1.txt（英文文本），2_2.txt（中文文本），2_3.bmp（图片），2_4.mp4（视频），2_5.exe（可执行文件）。

发现存在如下问题：

1. 解码时无法正常结束，无法确定末尾的0是否是补位产生的。

   解决方法：存储FileSize记录源文件大小，确定解码结束位置。
   
2. 解码时需要确定源文件类型。

   解决方法：将原文件类型以字符串形式存入.huff文件开头。
